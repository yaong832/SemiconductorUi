# 챔버 램프 로직 업데이트

**작성일:** 2025년 1월  
**문서 버전:** 2.1 (하드웨어 램프 깜빡임 추가)

## 1. 변경 사항 요약

챔버 램프 로직이 다음과 같이 변경되었습니다:

### 이전 로직
- 공정 중: 켜짐 (녹색)
- 그 외: 꺼짐 (밝은 회색)

### 새로운 로직
1. **공정 중**: 켜짐 (녹색)
2. **공정 완료 (웨이퍼 있음)**: 깜빡임 (녹색 ↔ 밝은 회색)
3. **웨이퍼 없음**: 꺼짐 (밝은 회색)

## 2. 구현 상세

### 2.1 상태 판단 메서드

#### IsChamberProcessing() - 공정 중 확인
```csharp
internal bool IsChamberProcessing(ChamberController.ChamberState chamber)
{
    // 기존 로직 유지
    // 조건: 웨이퍼 있음 + 공정 시간 설정됨 + 시간 남음 + 도어 닫힘 + 실제 공정 진행 상태
    return hasWafer && hasProcessTime && timeRemaining && doorClosed && isActuallyProcessing;
}
```

#### IsChamberCompleted() - 공정 완료 확인 (신규)
```csharp
internal bool IsChamberCompleted(ChamberController.ChamberState chamber)
{
    if (chamber == null) return false;
    
    bool hasWafer = chamber.CurrentWafer != null;
    bool hasProcessTime = chamber.TotalSeconds > 0;
    bool timeRemaining = chamber.RemainingSeconds <= 0;  // 공정 시간 없음
    bool doorClosed = !IsRegionDoorOpen(chamber.Region);
    
    return hasWafer && hasProcessTime && timeRemaining && doorClosed;
}
```

**조건:**
- 웨이퍼 있음 (`CurrentWafer != null`)
- 공정 시간 설정됨 (`TotalSeconds > 0`)
- 공정 시간 없음 (`RemainingSeconds <= 0`)
- 도어 닫힘 (`!IsRegionDoorOpen`)

### 2.2 램프 업데이트 로직

#### UpdateChamberProcessLamps()
```csharp
public void UpdateChamberProcessLamps()
{
    // 각 챔버의 상태 확인
    bool chamberAProcessing = form.IsChamberProcessing(form.ChamberAState);
    bool chamberBProcessing = form.IsChamberProcessing(form.ChamberBState);
    bool chamberCProcessing = form.IsChamberProcessing(form.ChamberCState);
    
    bool chamberACompleted = form.IsChamberCompleted(form.ChamberAState);
    bool chamberBCompleted = form.IsChamberCompleted(form.ChamberBState);
    bool chamberCCompleted = form.IsChamberCompleted(form.ChamberCState);

    // UI 램프 색상 및 깜빡임 업데이트
    UpdateChamberLampUI(form.panelLampChamberA, chamberAProcessing, chamberACompleted);
    UpdateChamberLampUI(form.panelLampChamberB, chamberBProcessing, chamberBCompleted);
    UpdateChamberLampUI(form.panelLampChamberC, chamberCProcessing, chamberCCompleted);

    // 실제 장비 램프 제어 (EtherCAT 연결 시)
    // 공정 중일 때만 켜고, 완료 상태나 웨이퍼 없을 때는 끔
    if (form.EthercatConnected && !form.isSyncingEquipmentState)
    {
        form.SetChamberLamp(EquipmentRegion.ChamberA, chamberAProcessing);
        form.SetChamberLamp(EquipmentRegion.ChamberB, chamberBProcessing);
        form.SetChamberLamp(EquipmentRegion.ChamberC, chamberCProcessing);
    }
}
```

### 2.3 깜빡임 구현

#### UpdateHardwareChamberLamp() - 하드웨어 램프 깜빡임 제어 (신규)
```csharp
private void UpdateHardwareChamberLamp(EquipmentRegion region, bool isProcessing, bool isCompleted)
{
    if (isProcessing)
    {
        // 공정 중: 램프 켜짐
        form.SetChamberLamp(region, true);
        // 깜빡임 상태 초기화
        ResetHardwareBlinkState(region);
    }
    else if (isCompleted)
    {
        // 공정 완료 (웨이퍼 있음): 깜빡임
        // 각 챔버별로 독립적인 깜빡임 카운터 사용
        if (!_hardwareLampBlinkCounters.ContainsKey(region))
        {
            _hardwareLampBlinkCounters[region] = 0;
            _hardwareLampBlinkStates[region] = false;
        }
        
        _hardwareLampBlinkCounters[region]++;
        if (_hardwareLampBlinkCounters[region] >= CHAMBER_LAMP_BLINK_INTERVAL)
        {
            _hardwareLampBlinkCounters[region] = 0;
            // 깜빡임 상태 토글
            _hardwareLampBlinkStates[region] = !_hardwareLampBlinkStates[region];
        }
        
        // 깜빡임 상태에 따라 하드웨어 램프 제어
        bool isBlinkOn = _hardwareLampBlinkStates[region];
        form.SetChamberLamp(region, isBlinkOn);
    }
    else
    {
        // 웨이퍼 없음: 램프 꺼짐
        form.SetChamberLamp(region, false);
        // 깜빡임 상태 초기화
        ResetHardwareBlinkState(region);
    }
}
```

#### UpdateChamberLampUI() - UI 깜빡임 로직 포함
```csharp
// 챔버 램프 깜빡임 상태 추적 (깜빡임 주기 제어용)
private readonly Dictionary<Panel, bool> _chamberLampBlinkStates = new Dictionary<Panel, bool>();
private readonly Dictionary<Panel, int> _chamberLampBlinkCounters = new Dictionary<Panel, int>();
private const int CHAMBER_LAMP_BLINK_INTERVAL = 10; // 10틱마다 깜빡임

public void UpdateChamberLampUI(Panel lampPanel, bool isProcessing, bool isCompleted)
{
    if (lampPanel == null) return;
    
    if (isProcessing)
    {
        // 공정 중: 녹색 켜짐
        lampPanel.BackColor = Color.ForestGreen;
        // 깜빡임 상태 초기화
        ResetBlinkState(lampPanel);
    }
    else if (isCompleted)
    {
        // 공정 완료 (웨이퍼 있음): 깜빡임
        // 각 챔버별로 독립적인 깜빡임 카운터 사용
        if (!_chamberLampBlinkCounters.ContainsKey(lampPanel))
        {
            _chamberLampBlinkCounters[lampPanel] = 0;
            _chamberLampBlinkStates[lampPanel] = false;
        }
        
        _chamberLampBlinkCounters[lampPanel]++;
        if (_chamberLampBlinkCounters[lampPanel] >= CHAMBER_LAMP_BLINK_INTERVAL)
        {
            _chamberLampBlinkCounters[lampPanel] = 0;
            // 깜빡임 상태 토글
            _chamberLampBlinkStates[lampPanel] = !_chamberLampBlinkStates[lampPanel];
        }
        
        // 깜빡임 상태에 따라 색상 변경
        bool isBlinkOn = _chamberLampBlinkStates[lampPanel];
        lampPanel.BackColor = isBlinkOn ? Color.ForestGreen : Color.FromArgb(230, 230, 235);
    }
    else
    {
        // 웨이퍼 없음: 꺼짐 (밝은 회색)
        lampPanel.BackColor = Color.FromArgb(230, 230, 235);
        // 깜빡임 상태 초기화
        ResetBlinkState(lampPanel);
    }
}
```

### 2.4 깜빡임 특징

- **독립적인 깜빡임**: 각 챔버별로 독립적인 깜빡임 카운터 사용 (UI 및 하드웨어 모두)
- **깜빡임 주기**: 10틱마다 토글 (시뮬레이션 타이머 기준)
- **UI 깜빡임 색상**: 녹색 ↔ 밝은 회색
- **하드웨어 깜빡임**: ON ↔ OFF (실제 장비 램프 제어)
- **동기화**: UI와 하드웨어 램프가 동일한 주기로 깜빡임

## 3. 상태 전환 시나리오

### 3.1 공정 시작 → 공정 중

```
1. 웨이퍼가 챔버에 할당됨
   ↓
2. 공정 시간 설정됨
   ↓
3. 도어가 닫힘
   ↓
4. StatusText가 "처리 중"으로 변경됨
   ↓
5. IsChamberProcessing() = true
   ↓
6. 램프: 녹색 켜짐 (깜빡임 없음)
```

### 3.2 공정 중 → 공정 완료

```
1. RemainingSeconds가 0이 됨
   ↓
2. StatusText가 "Completed"로 변경됨
   ↓
3. IsChamberProcessing() = false
   ↓
4. IsChamberCompleted() = true
   ↓
5. 램프: 깜빡임 시작
   - UI 램프: 녹색 ↔ 회색 깜빡임
   - 하드웨어 램프: ON ↔ OFF 깜빡임
```

### 3.3 공정 완료 → 웨이퍼 제거

```
1. 웨이퍼가 챔버에서 제거됨
   ↓
2. CurrentWafer = null
   ↓
3. IsChamberCompleted() = false
   ↓
4. 램프: 꺼짐 (깜빡임 중지)
   - UI 램프: 밝은 회색
   - 하드웨어 램프: OFF
```

## 4. 하드웨어 제어

### 4.1 하드웨어 램프 동작

- **공정 중**: 하드웨어 램프 ON
- **공정 완료**: 하드웨어 램프 깜빡임 (ON ↔ OFF)
- **웨이퍼 없음**: 하드웨어 램프 OFF

**구현 방식:**
- 각 챔버별로 독립적인 깜빡임 카운터 사용
- 시뮬레이션 타이머 주기(10틱)마다 하드웨어 램프 상태 토글
- `UpdateHardwareChamberLamp()` 메서드에서 하드웨어 제어 수행

1. 웨이퍼 로드 → 램프 꺼짐
