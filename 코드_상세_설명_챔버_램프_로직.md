# 챔버 램프 로직 상세 설명

**작성일:** 2025년 1월  
**문서 버전:** 1.0

---

## 1. 개요

챔버 램프는 각 챔버(Chamber A, B, C)의 **공정 진행 상태**를 시각적으로 표시하는 램프입니다. 공정이 진행 중일 때는 **녹색(ForestGreen)**으로 켜지고, 대기 상태일 때는 **꺼집니다**.

### 1.1 주요 특징

- **공정 상태 표시**: 도어 상태가 아닌 **실제 공정 진행 상태**를 표시
- **이중 제어**: UI 램프와 실제 하드웨어 램프를 동시에 제어
- **조건 기반**: 여러 조건을 모두 만족해야 램프가 켜짐

---

## 2. 램프 켜짐 조건

### 2.1 IsChamberProcessing() 메서드

챔버 램프가 켜지기 위해서는 다음 **5가지 조건을 모두 만족**해야 합니다:

```csharp
internal bool IsChamberProcessing(ChamberController.ChamberState chamber)
{
    if (chamber == null) return false;
    
    // 조건 1: 웨이퍼가 있어야 함
    bool hasWafer = chamber.CurrentWafer != null;
    
    // 조건 2: 공정 시간이 설정되어 있어야 함
    bool hasProcessTime = chamber.TotalSeconds > 0;
    
    // 조건 3: 공정 시간이 남아있어야 함 (프로그레스바가 움직이는 조건)
    bool timeRemaining = chamber.RemainingSeconds > 0;
    
    // 조건 4: 도어가 닫혀있어야 함
    bool doorClosed = !IsRegionDoorOpen(chamber.Region);
    
    // 조건 5: 실제 공정 진행 중이어야 함
    // StatusText가 "처리 중", "2차 노광 중", "Processing" 중 하나여야 함
    // "Door Open 대기", "Door Closing", "TM 대기" 등의 상태에서는 false
    bool isActuallyProcessing = !string.IsNullOrEmpty(chamber.StatusText) &&
        (chamber.StatusText.Contains("처리 중") || 
         chamber.StatusText.Contains("2차 노광 중") ||
         chamber.StatusText.Contains("Processing"));
    
    // 모든 조건을 만족해야 true
    return hasWafer && hasProcessTime && timeRemaining && doorClosed && isActuallyProcessing;
}
```

### 2.2 조건 상세 설명

#### 조건 1: 웨이퍼 존재 (`hasWafer`)
- 챔버에 웨이퍼가 할당되어 있어야 함
- `chamber.CurrentWafer != null`

#### 조건 2: 공정 시간 설정 (`hasProcessTime`)
- 공정 시간이 설정되어 있어야 함
- `chamber.TotalSeconds > 0`

#### 조건 3: 공정 시간 남음 (`timeRemaining`)
- 공정 시간이 남아있어야 함
- `chamber.RemainingSeconds > 0`
- 프로그레스바가 움직이는 조건과 동일

#### 조건 4: 도어 닫힘 (`doorClosed`)
- 챔버 도어가 닫혀있어야 함
- `!IsRegionDoorOpen(chamber.Region)`

#### 조건 5: 실제 공정 진행 중 (`isActuallyProcessing`)
- StatusText가 다음 중 하나여야 함:
  - `"처리 중"`
  - `"2차 노광 중"`
  - `"Processing"`
- 다음 상태에서는 **램프가 꺼짐**:
  - `"Door Open 대기"`
  - `"Door Closing"`
  - `"TM 대기"`
  - `"Idle"`
  - `"Completed"`

## 3. 램프 업데이트 흐름

### 3.1 UpdateChamberProcessLamps() 메서드

챔버 램프는 `UpdateChamberProcessLamps()` 메서드에서 업데이트됩니다:

```csharp
public void UpdateChamberProcessLamps()
{
    // 각 챔버의 공정 상태 확인
    bool chamberAProcessing = form.IsChamberProcessing(form.ChamberAState);
    bool chamberBProcessing = form.IsChamberProcessing(form.ChamberBState);
    bool chamberCProcessing = form.IsChamberProcessing(form.ChamberCState);

    // UI 램프 색상 업데이트
    UpdateChamberLampUI(form.panelLampChamberA, chamberAProcessing);
    UpdateChamberLampUI(form.panelLampChamberB, chamberBProcessing);
    UpdateChamberLampUI(form.panelLampChamberC, chamberCProcessing);

    // 실제 장비 램프 제어 (EtherCAT 연결 시)
    if (form.EthercatConnected && !form.isSyncingEquipmentState)
    {
        form.SetChamberLamp(EquipmentRegion.ChamberA, chamberAProcessing);
        form.SetChamberLamp(EquipmentRegion.ChamberB, chamberBProcessing);
        form.SetChamberLamp(EquipmentRegion.ChamberC, chamberCProcessing);
    }
}
```

### 3.2 호출 시점

`UpdateChamberProcessLamps()`는 다음 시점에 호출됩니다:

1. **시뮬레이션 타이머 Tick**: 공정 진행 중 주기적으로 호출
2. **UI 업데이트**: `UpdateSimulationUi()` 메서드에서 호출
3. **상태 변경 시**: 챔버 상태가 변경될 때 호출

## 4. UI 램프 업데이트

### 4.1 UpdateChamberLampUI() 메서드

UI 램프의 색상을 업데이트합니다:

```csharp
public void UpdateChamberLampUI(Panel lampPanel, bool isProcessing)
{
    if (lampPanel == null) return;
    
    // 공정 중: 녹색(ForestGreen), 대기: 밝은 회색
    lampPanel.BackColor = isProcessing 
        ? Color.ForestGreen      // 공정 중: 녹색
        : Color.FromArgb(230, 230, 235);  // 대기: 밝은 회색
}
```

### 4.2 램프 색상

- **공정 중**: `Color.ForestGreen` (녹색)
- **대기**: `Color.FromArgb(230, 230, 235)` (밝은 회색)

## 5. 하드웨어 램프 제어

### 5.1 SetChamberLamp() 메서드

실제 장비의 램프를 제어합니다:

```csharp
internal void SetChamberLamp(EquipmentRegion region, bool on)
{
    if (!EthercatConnected)
    {
        return;
    }

    try
    {
        if (!ChamberHardwareHelper.ControlChamberLamp(EtherCAT_M, region, on))
        {
            AddLogMessage($"{EquipmentRegionHelper.FormatRegionLabel(region)} 램프 제어 실패", "ERROR");
        }
    }
    catch (Exception ex)
    {
        AddLogMessage($"{EquipmentRegionHelper.FormatRegionLabel(region)} 램프 제어 오류: {ex.Message}", "ERROR");
    }
}
```

### 5.2 ControlChamberLamp() 메서드

`ChamberHardwareHelper.ControlChamberLamp()` 메서드가 실제 하드웨어를 제어합니다:

```csharp
public static bool ControlChamberLamp(IEG3268 ethercatDevice, EquipmentRegion region, bool on)
{
    if (ethercatDevice == null)
    {
        return false;
    }

    // 챔버별 I/O 인덱스 가져오기
    int outputIndex = GetChamberLampIoIndex(region);
    if (outputIndex < 0)
    {
        return false;
    }

    try
    {
        // EtherCAT Digital Output으로 램프 제어
        ethercatDevice.Digital_Output(outputIndex, on);
        return true;
    }
    catch
    {
        return false;
    }
}
```

### 5.3 I/O 인덱스 매핑

각 챔버의 램프는 다음 Digital Output 인덱스를 사용합니다:

- **Chamber A**: Digital Output **3**
- **Chamber B**: Digital Output **6**
- **Chamber C**: Digital Output **9**

## 6. 하드웨어 상태 동기화

### 6.1 SyncEquipmentStateToUI() 메서드

EtherCAT 연결 시, 실제 장비의 램프 상태를 읽어서 UI에 반영합니다:

```csharp
internal void SyncEquipmentStateToUI()
{
    if (!EthercatConnected || EtherCAT_M == null)
    {
        return;
    }

    isSyncingEquipmentState = true; // 동기화 시작
    try
    {
        System.Threading.Thread.Sleep(200); // 200ms 대기
        
        // Chamber A, B, C에 대한 도어 및 램프 상태 매핑
        var chamberMappings = new[]
        {
            (EquipmentRegion.ChamberA, 5, 3),  // (Region, 도어 Input, 램프 Input)
            (EquipmentRegion.ChamberB, 8, 6),
            (EquipmentRegion.ChamberC, 11, 9)
        };

        // 실제 장비의 램프 상태 읽기 및 UI 반영
        foreach (var (region, doorInputIndex, lampInputIndex) in chamberMappings)
        {
            bool lampOn = false;
            
            try
            {
                // IEG3268_Dll의 Digital_Input 메서드를 사용하여 실제 장비 상태 읽기
                lampOn = EtherCAT_M.Digital_Input(lampInputIndex);
            }
            catch (Exception ex)
            {
                AddLogMessage($"장비 상태 읽기 오류 ({region}): {ex.Message}", "WARN");
                continue;
            }

            // 읽은 램프 상태를 UI에 반영 (장비 제어는 하지 않고 UI만 업데이트)
            SetChamberLampUIOnly(region, lampOn);
        }
    }
    finally
    {
        isSyncingEquipmentState = false; // 동기화 종료
    }
}
```

### 6.2 I/O 인덱스 매핑 (읽기)

각 챔버의 램프 상태는 다음 Digital Input 인덱스에서 읽습니다:

- **Chamber A**: Digital Input **3**
- **Chamber B**: Digital Input **6**
- **Chamber C**: Digital Input **9**

### 6.3 SetChamberLampUIOnly() 메서드

동기화 시에는 장비를 제어하지 않고 UI만 업데이트합니다:

```csharp
internal void SetChamberLampUIOnly(EquipmentRegion region, bool on)
{
    Panel lampPanel = null;
    switch (region)
    {
        case EquipmentRegion.ChamberA:
            lampPanel = panelLampChamberA;
            break;
        case EquipmentRegion.ChamberB:
            lampPanel = panelLampChamberB;
            break;
        case EquipmentRegion.ChamberC:
            lampPanel = panelLampChamberC;
            break;
    }

    if (lampPanel != null)
    {
        SetLamp(lampPanel, on, Color.Yellow);  // 동기화 시에는 황색으로 표시
    }
}
```

## 7. 램프 상태 전환 시나리오

### 7.1 공정 시작 시

```
1. 웨이퍼가 챔버에 할당됨 (CurrentWafer != null)
   ↓
2. 공정 시간 설정됨 (TotalSeconds > 0)
   ↓
3. 도어가 닫힘 (doorClosed = true)
   ↓
4. StatusText가 "처리 중"으로 변경됨
   ↓
5. IsChamberProcessing() = true
   ↓
6. UpdateChamberProcessLamps() 호출
   ↓
7. UI 램프: 녹색(ForestGreen)으로 변경
   ↓
8. 하드웨어 램프: ON (EtherCAT 연결 시)
```

### 7.2 공정 완료 시

```
1. RemainingSeconds가 0이 됨
   ↓
2. StatusText가 "Completed"로 변경됨
   ↓
3. IsChamberProcessing() = false
   ↓
4. UpdateChamberProcessLamps() 호출
   ↓
5. UI 램프: 밝은 회색으로 변경
   ↓
6. 하드웨어 램프: OFF (EtherCAT 연결 시)
```

### 7.3 도어 열림 시

```
1. 도어가 열림 (doorClosed = false)
   ↓
2. IsChamberProcessing() = false (조건 4 불만족)
   ↓
3. UpdateChamberProcessLamps() 호출
   ↓
4. UI 램프: 밝은 회색으로 변경
   ↓
5. 하드웨어 램프: OFF (EtherCAT 연결 시)
```

## 8. 주의사항

### 8.1 동기화 중 제어 방지

`isSyncingEquipmentState` 플래그가 `true`일 때는 하드웨어 램프를 제어하지 않습니다:

```csharp
// 실제 장비 램프 제어 (EtherCAT 연결 시)
if (form.EthercatConnected && !form.isSyncingEquipmentState)
{
    form.SetChamberLamp(EquipmentRegion.ChamberA, chamberAProcessing);
    // ...
}
```

이렇게 하면 동기화 중에 램프가 깜빡이거나 잘못된 상태로 변경되는 것을 방지합니다.

### 8.2 StatusText 조건

StatusText가 정확히 일치해야 램프가 켜집니다:

- ✅ **켜짐**: `"처리 중"`, `"2차 노광 중"`, `"Processing"`
- ❌ **꺼짐**: `"Door Open 대기"`, `"Door Closing"`, `"TM 대기"`, `"Idle"`, `"Completed"`

## 9. 코드 흐름도

```
UpdateSimulationUi()
    ↓
UpdateChamberProcessLamps()
    ↓
IsChamberProcessing(ChamberAState)
    ├─ hasWafer? (CurrentWafer != null)
    ├─ hasProcessTime? (TotalSeconds > 0)
    ├─ timeRemaining? (RemainingSeconds > 0)
    ├─ doorClosed? (!IsRegionDoorOpen)
    └─ isActuallyProcessing? (StatusText 확인)
    ↓
true → UpdateChamberLampUI(녹색)
false → UpdateChamberLampUI(회색)
    ↓
EthercatConnected && !isSyncingEquipmentState?
    ├─ Yes → SetChamberLamp(하드웨어 제어)
    └─ No → UI만 업데이트
```

## 10. 요약

### 10.1 램프 켜짐 조건

챔버 램프가 켜지기 위해서는 다음 **5가지 조건을 모두 만족**해야 합니다:

1. ✅ 웨이퍼 존재 (`CurrentWafer != null`)
2. ✅ 공정 시간 설정 (`TotalSeconds > 0`)
3. ✅ 공정 시간 남음 (`RemainingSeconds > 0`)
4. ✅ 도어 닫힘 (`!IsRegionDoorOpen`)
5. ✅ 실제 공정 진행 중 (`StatusText` 확인)

### 10.2 램프 색상

- **공정 중**: 녹색 (`Color.ForestGreen`)
- **대기**: 밝은 회색 (`Color.FromArgb(230, 230, 235)`)

### 10.3 하드웨어 제어

- **Chamber A**: Digital Output 3
- **Chamber B**: Digital Output 6
- **Chamber C**: Digital Output 9

### 10.4 주요 특징

- ✅ 공정 상태 기반 (도어 상태 아님)
- ✅ UI와 하드웨어 동시 제어
- ✅ 동기화 중 제어 방지
- ✅ 조건 기반 정확한 상태 표시
